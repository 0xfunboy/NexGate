NEXUS CONTROL API

Playwright-driven gateway over ChatGPT, Claude, Gemini, and Grok web sessions.

Base URL

http://localhost:3001

Core Endpoints

Health and provider state

  • GET /health basic liveness and provider ids
  • GET /providers provider config plus accounts
  • GET /providers/overview provider status summary for the dashboard
  • GET /providers/:provider/status detailed status for one provider

Session control

  • POST /providers/:provider/login open or restore provider login in Playwright
  • POST /providers/:provider/reset close current provider tab
  • POST /providers/:provider/accounts/rotate rotate to the next account
  • POST /providers/:provider/accounts/reset-quota clear exhausted flags

Prompt execution

  • POST /providers/:provider/prompt synchronous prompt execution
  • POST /providers/:provider/stream SSE token stream for one provider
  • POST /compare synchronous multi-provider compare
  • POST /compare/stream NDJSON multi-provider streaming compare

Audio

  • POST /providers/grok/audio capture Grok read-aloud output
  • GET /providers/grok/audio/latest retrieve the last captured audio

OpenAI-Compatible

GET /v1/models

Returns one synthetic model per provider, for example gemini:frontend-default.

GET /v1/models/:model returns a single model object for SDKs that probe individual model ids.

POST /v1/chat/completions

Accepts OpenAI-style chat messages, including developer role and array content parts. Supports stream: true with SSE.

{
  "model": "chatgpt:frontend-default",
  "messages": [
    { "role": "developer", "content": "Answer in concise Italian." },
    { "role": "user", "content": "Write a haiku about debugging" }
  ],
  "stream": false
}

POST /v1/responses

Accepts a string input or message array. instructions is mapped to a system message. Supports streaming with OpenAI Responses-style SSE events.

{
  "model": "gemini:frontend-default",
  "instructions": "Answer like a senior engineer.",
  "input": "Explain the adapter pattern",
  "stream": true
}

Ollama-Compatible

POST /api/generate

Generate-style endpoint with optional streaming.

POST /api/chat

Chat-style endpoint with message arrays and optional streaming.

Notes

  • The gateway uses real consumer web sessions. Login and quota behavior come from the provider websites.
  • Extra fields such as images may be present on some responses when the provider generated media.
  • Chat and Responses payloads tolerate extra OpenAI SDK fields that this gateway ignores.
  • Usage numbers are estimated, not tokenizer-accurate.
  • Requests for the same provider are serialized to avoid Playwright input collisions.
